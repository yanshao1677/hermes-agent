# 用导入时自注册和可用性检查实现工具注册表

## 1. 问题

面向模型的工具系统同时需要暴露工具规格、执行工具处理器、按场景启用工具、隐藏不可用工具。若把 schema 列表、处理器分发表、依赖检查表分别维护，会出现三类问题：新增工具漏改某个表、模型看到不可用工具、工具描述提到当前未启用的其它工具。

## 2. 适用约束

- 工具由多个独立模块提供，且存在可选依赖或外部凭据。
- 调用模型前必须生成符合函数调用协议的工具 schema。
- 不同入口或不同子任务需要不同工具范围。
- 工具处理器需要统一返回可序列化字符串，便于写回模型上下文。

## 3. 核心思路

让每个工具模块在导入时把 `name / toolset / schema / handler / check_fn / requires_env` 注册到中央注册表；运行时先解析工具集，再由注册表按可用性函数过滤 schema，最后只对实际可用工具做动态描述裁剪。

## 4. 通用结构

- **ToolEntry**：单个工具元数据，包含工具名、所属工具集、模型可见 schema、执行处理器、可用性检查函数、环境变量要求、是否异步、结果大小限制。
- **ToolRegistry**：进程内单例，提供注册、反注册、schema 获取、分发执行、工具集可用性检查。
- **ToolDiscovery**：导入工具模块的阶段；导入副作用就是注册。
- **ToolsetResolver**：把高层工具集名解析成工具名集合，支持组合工具集。
- **SchemaPostProcessor**：基于“实际通过检查的工具名集合”重写部分 schema 描述，避免描述引用不存在工具。

## 5. 处理流程

1. 启动时导入中央注册表。
2. 逐个导入工具模块，模块顶层调用注册表的注册方法。
3. 调用方传入启用工具集或禁用工具集。
4. 解析工具集为目标工具名集合。
5. 注册表遍历目标工具名，执行每个工具的可用性函数；同一个检查函数结果在一次 schema 获取中缓存。
6. 只返回通过检查的工具 schema。
7. 计算实际可用工具名集合。
8. 对会引用其它工具名的 schema 做动态裁剪。
9. 执行工具时统一通过注册表按名称分发；未知工具返回结构化错误。

## 6. 异常处理

- 导入可选工具模块失败：记录失败，不影响其它工具注册。
- 可用性函数抛异常：该工具视为不可用。
- 分发未知工具名：返回 `{"error": "Unknown tool: <name>"}`。
- 工具处理器抛异常：捕获并返回 `{"error": "Tool execution failed: <异常类型>: <消息>"}`。

## 7. 具体语言实现

```python
import json
from dataclasses import dataclass
from typing import Callable

@dataclass
class ToolEntry:
    name: str
    toolset: str
    schema: dict
    handler: Callable[[dict], str]
    check_fn: Callable[[], bool] | None = None
    requires_env: tuple[str, ...] = ()

class ToolRegistry:
    def __init__(self):
        self._tools: dict[str, ToolEntry] = {}

    def register(self, entry: ToolEntry):
        self._tools[entry.name] = entry

    def definitions(self, names: set[str]) -> list[dict]:
        cache: dict[Callable[[], bool], bool] = {}
        result = []
        for name in sorted(names):
            entry = self._tools.get(name)
            if not entry:
                continue
            if entry.check_fn:
                if entry.check_fn not in cache:
                    try:
                        cache[entry.check_fn] = bool(entry.check_fn())
                    except Exception:
                        cache[entry.check_fn] = False
                if not cache[entry.check_fn]:
                    continue
            schema = {**entry.schema, "name": entry.name}
            result.append({"type": "function", "function": schema})
        return result

    def dispatch(self, name: str, args: dict) -> str:
        entry = self._tools.get(name)
        if not entry:
            return json.dumps({"error": f"Unknown tool: {name}"})
        try:
            return entry.handler(args)
        except Exception as exc:
            return json.dumps({"error": f"Tool execution failed: {type(exc).__name__}: {exc}"})
```

## 8. 测试点

- 注册工具后能按名称返回 schema 和分发执行。
- 可用性函数返回 False 时 schema 不暴露。
- 可用性函数抛异常时 schema 不暴露。
- 未知工具执行返回结构化错误。
- schema 动态裁剪只引用实际可用工具。

## 9. 适用场景 / 不适用场景

适用于单进程内生成模型工具清单、工具模块较多、外部依赖可选的智能体系统。不适用于需要多进程动态注册立即同步的工具市场；那类系统应把注册表放到数据库或服务发现层。

## 10. 风险与反模式

- 导入副作用让注册顺序变得重要；必须集中维护发现列表或插件发现机制。
- 进程内注册表不能表达跨进程一致性。
- schema 描述写死其它工具名会导致模型调用不存在工具；应在 schema 后处理阶段根据实际可用集合裁剪。
