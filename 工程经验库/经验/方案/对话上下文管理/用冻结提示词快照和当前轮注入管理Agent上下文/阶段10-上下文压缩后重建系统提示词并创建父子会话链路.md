# 阶段10：上下文压缩后重建系统提示词并创建父子会话链路

## 触发条件

由阶段 8（预压缩）、阶段 11（API 错误重试）或主循环 token 压力（`should_compress(real_tokens)`）调用 `_compress_context()`。需 `compression_enabled` 且消息规模超过压缩器最小要求。

## 输入字段

- `messages`：待压缩真实历史。
- `system_message`：重建 system 时的可选补充。
- `approx_tokens`：可选估算 token。
- `focus_topic`：可选压缩聚焦主题。
- `context_compressor`：含 `threshold_tokens`、`protect_first_n`、`protect_last_n`、`compression_count`。
- `session_db`、`session_id`。

## 判定规则

1. 压缩前 `flush_memories(messages, min_turns=0)`（不受普通 min_turns 限制）。
2. 外部记忆 `on_pre_compress(messages)`；失败跳过。
3. `compressed = context_compressor.compress(messages, current_tokens=approx_tokens, focus_topic=focus_topic)`。
4. 若有 todo 快照 → 追加为一条 `user` 消息。
5. `_invalidate_system_prompt()` + `_build_system_prompt()` → 新 `cached_system_prompt`。
6. 有 `session_db` 时：
   - `end_session(old_id, "compression")`
   - 新 `session_id = YYYYMMDD_HHMMSS_{uuid6}`
   - `create_session(..., parent_session_id=old_id)`
   - 迁移 title lineage（可选）
   - `update_system_prompt(new_id, new_prompt)`
   - `last_flushed_db_idx = 0`
7. `compression_count >= 2` → 向用户建议 `/new` 开新对话。

**历史会话搜索**依赖本阶段写入的 `parent_session_id` 链路；FTS 排除当前 lineage 的细节见记忆域 [阶段2-用FTS检索并按父链路聚合历史会话](../../记忆与召回/用全文检索和摘要实现历史会话情景召回/阶段2-用FTS检索并按父链路聚合历史会话.md)，不在本卡重复。

## 状态读写位置

- 读写：真实历史、system 缓存、本地记忆文件。
- 写：新 session_id、session 日志路径、parent 链路、flush 游标归零。
- 返回：`(compressed_messages, new_system_prompt)`。

## 正常路径

1. flush memory → on_pre_compress → compress。
2. 追加 todo 快照（若有）。
3. 失效并重建 system prompt。
4. DB 拆分 session，重置游标。
5. 调用方清空 `conversation_history`（阶段 8/11 负责）。

## 分支路径

- 无 session_db → 只压缩 + 重建 prompt，无父子链路。
- todo 为空 → 不追加。
- DB 拆分失败 → warning；压缩后消息仍可用但新 session 可能不可索引。
- 压缩后消息数未减且 context 已最小 → 阶段 11 可能终止重试。

## 失败处理

`on_pre_compress` 与 DB 拆分失败均不阻断压缩结果返回。压缩器内部摘要失败分支证据不足，本卡只规定外层契约。

## 幂等性 / 一致性约束

每次成功压缩产生新 session 与新 system 快照；不可当作幂等操作。压缩后同一进程内后续 API 调用必须使用新 `cached_system_prompt`。续会话恢复快照的例外见阶段 2；压缩是显式刷新例外。

## 代码骨架

```python
def compress_context(messages, state, system_message=None, approx_tokens=None, focus_topic=None):
    state.flush_memories(messages, min_turns=0)
    if state.memory_manager:
        try:
            state.memory_manager.on_pre_compress(messages)
        except Exception:
            pass
    compressed = state.compressor.compress(messages, current_tokens=approx_tokens, focus_topic=focus_topic)
    if todo := state.todo_store.format_for_injection():
        compressed.append({"role": "user", "content": todo})
    state.cached_system_prompt = None
    state.memory_store.load_from_disk()
    new_prompt = build_system_prompt(system_message)
    state.cached_system_prompt = new_prompt
    if state.session_db:
        old_id = state.session_id
        state.session_db.end_session(old_id, "compression")
        state.session_id = new_session_id()
        state.session_db.create_session(state.session_id, parent_session_id=old_id, ...)
        state.session_db.update_system_prompt(state.session_id, new_prompt)
        state.last_flushed_db_idx = 0
    return compressed, new_prompt
```

## 最小验证清单

1. 压缩前 `flush_memories(min_turns=0)`。
2. 压缩后 `cached_system_prompt` 先 None 再重建。
3. 新 session 的 `parent_session_id ==` 旧 session_id。
4. `last_flushed_db_idx == 0`。
5. 第 2 次压缩后用户可见“建议 /new”提示。
