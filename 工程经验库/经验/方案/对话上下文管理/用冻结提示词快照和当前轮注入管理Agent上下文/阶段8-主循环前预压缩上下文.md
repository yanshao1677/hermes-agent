# 阶段8：主循环前预压缩上下文

## 触发条件

系统提示词快照就绪后、进入主工具循环**之前**。需同时满足：

1. `compression_enabled == True`
2. `len(messages) > protect_first_n + protect_last_n + 1`
3. `estimate_request_tokens_rough(messages, system_prompt, tools)` ≥ `threshold_tokens`

## 输入字段

| 字段 | 说明 |
|------|------|
| `messages` | 真实历史（含当前 user） |
| `active_system_prompt` | 稳定 system 快照 |
| `self.tools` | 工具 schema 列表（计入 token 估算） |
| `conversation_history` | 传入的旧历史引用（可被清空） |
| `context_compressor` | 含 `threshold_tokens`、`protect_first_n`、`protect_last_n` |

## 判定规则

### R1：token 估算

预检必须计入 **tools schema token**（注释：多工具时可占 2–3 万 token），不能只算 messages + system。

### R2：多轮压缩循环

最多 **3** 次 pass：

```text
for pass in range(3):
    orig_len = len(messages)
    messages, active_system_prompt = compress_context(...)
    if len(messages) >= orig_len: break  # 无法再压缩
    conversation_history = None  # 压缩产生新 session，必须清空
    re-estimate tokens
    if tokens < threshold_tokens: break
```

### R3：conversation_history 清空

每次压缩成功后 **`conversation_history = None`**，确保 `_flush_messages_to_session_db` 的 `start_idx=0`，把压缩后消息完整写入新 session（不因旧 `conversation_history` 长度跳过）。

### R4：压缩实现

调用阶段 10 的 `_compress_context()`（flush memory、on_pre_compress、compress、invalidate、split session）。

## 状态读写位置

- 读写：`messages`、`active_system_prompt`、`session_id`（压缩拆分时变更）。
- 写：`conversation_history` 引用置 `None`。
- 不写：API 请求副本。

## 正常路径

1. 估算含 tools 的 token。
2. 若 ≥ 阈值，打印预压缩日志（非 quiet 模式）。
3. 执行最多 3 轮 compress + 重估。
4. 进入阶段 3（收集临时上下文）与主循环。

## 分支路径

- 未达阈值 → 跳过本阶段。
- 压缩后消息数未减少 → 终止 pass 循环。
- 第 1 次 pass 后 token 已低于阈值 → 不再继续 pass。
- `compression_enabled == False` → 跳过。

## 失败处理

单次 `_compress_context` 内部失败策略见阶段 10；预检循环不因单次 pass 无进展而抛错，仅 break。

## 幂等性 / 一致性约束

- 预压缩与主循环内压缩（阶段 10/11）共用同一 `_compress_context`，均会产生新 session 与新建 system 快照。
- 预压缩后必须清空 `conversation_history`，与阶段 11 压缩后行为一致。
- 预压缩**不**收集临时上下文；prefetch 在压缩**之后**执行（阶段 3）。

## 代码骨架

```python
def preflight_compress_if_needed(messages, active_system_prompt, tools, state, history_ref):
    if not state.compression_enabled:
        return messages, active_system_prompt, history_ref
    min_msgs = state.compressor.protect_first_n + state.compressor.protect_last_n + 1
    if len(messages) <= min_msgs:
        return messages, active_system_prompt, history_ref
    tokens = estimate_request_tokens_rough(messages, active_system_prompt, tools)
    if tokens < state.compressor.threshold_tokens:
        return messages, active_system_prompt, history_ref
    for _ in range(3):
        orig = len(messages)
        messages, active_system_prompt = compress_context(messages, ...)
        if len(messages) >= orig:
            break
        history_ref = None
        tokens = estimate_request_tokens_rough(messages, active_system_prompt, tools)
        if tokens < state.compressor.threshold_tokens:
            break
    return messages, active_system_prompt, history_ref
```

## 最小验证清单

1. 切换至更小 context 窗口的大会话，进入主循环前触发预压缩而非等 API 400。
2. token 估算包含 tools 项。
3. 压缩后 `conversation_history is None`。
4. 最多 3 次 pass；消息数未减则停止。
5. 预压缩发生在 `prefetch_all` 之前。
