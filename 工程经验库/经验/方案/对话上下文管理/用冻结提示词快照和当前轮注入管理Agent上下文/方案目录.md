# 方案目录：用冻结提示词快照和当前轮注入管理 Agent 上下文

> 本方案是架构 ADR《用稳定系统提示词与临时上下文分层》的落地流水线。外部记忆 prefetch/sync 细节见记忆域「提供者编排层」方案；历史会话 FTS 召回见记忆域「全文检索和摘要」方案。

## 所属方案

- [用冻结提示词快照和当前轮注入管理Agent上下文](../用冻结提示词快照和当前轮注入管理Agent上下文.md)

## 阶段顺序

1. [阶段1-当前轮临时上下文只注入用户消息副本](阶段1-当前轮临时上下文只注入用户消息副本.md)
   - 解决问题：外部记忆 prefetch 与插件上下文只追加到当前轮 user 的 API 副本，不污染真实历史或 system prompt。
   - 衔接约束：输出为无 system 的历史副本；交给阶段 4 插入 system/prefill。
2. [阶段2-续会话优先恢复系统提示词快照而不是重新构建](阶段2-续会话优先恢复系统提示词快照而不是重新构建.md)
   - 解决问题：跨进程续会话时优先恢复已保存 system prompt，避免重建导致稳定前缀漂移。
   - 衔接约束：压缩（阶段 10）是显式刷新快照的例外。
3. [阶段7-API用户消息与持久化用户消息分离](阶段7-API用户消息与持久化用户消息分离.md)
   - 解决问题：gateway/技能路径下 API user 与 DB 存盘 user 可不同；预取查询词用清洗版。
   - 衔接约束：override 只在阶段 9 persist 前应用；API 构造仍用 `user_message`。
4. [阶段8-主循环前预压缩上下文](阶段8-主循环前预压缩上下文.md)
   - 解决问题：进入主循环前主动压缩超大历史，避免先撞 API 4xx。
   - 衔接约束：压缩后必须 `conversation_history=None`；发生在阶段 3 prefetch **之前**。
5. [阶段3-主循环前收集临时上下文并在工具循环中复用](阶段3-主循环前收集临时上下文并在工具循环中复用.md)
   - 解决问题：插件与外部记忆在本轮只收集一次，工具循环内复用。
   - 衔接约束：收集失败降级为空串；查询词用阶段 7 的 `original_user_message`。
6. [阶段4-系统消息拼接与预填充消息插入请求副本](阶段4-系统消息拼接与预填充消息插入请求副本.md)
   - 解决问题：stable system + ephemeral system + prefill 的请求顺序。
   - 衔接约束：插件 context 不得进入 system；只写 API 副本。
7. [阶段5-Anthropic提示词缓存断点策略](阶段5-Anthropic提示词缓存断点策略.md)
   - 解决问题：`system_and_3` 策略下最多 4 个 cache_control 断点。
   - 衔接约束：在阶段 6 归一化之前执行。
8. [阶段6-请求副本归一化与协议清理](阶段6-请求副本归一化与协议清理.md)
   - 解决问题：sanitize 配对、content strip、tool args sort_keys。
   - 衔接约束：与工具容错阶段 1 共用 `_sanitize_api_messages`；只改 API 副本。
9. [阶段9-只持久化真实历史并用刷新索引避免重复写入](阶段9-只持久化真实历史并用刷新索引避免重复写入.md)
   - 解决问题：多退出路径 persist 不重复写 DB。
   - 衔接约束：压缩新 session 时游标须为 0（阶段 10）。
10. [阶段10-上下文压缩后重建系统提示词并创建父子会话链路](阶段10-上下文压缩后重建系统提示词并创建父子会话链路.md)
    - 解决问题：compress 后 flush memory、重建 snapshot、拆分 session、`parent_session_id`。
    - 衔接约束：被阶段 8、11 及主循环 token 压力调用；FTS lineage 见记忆域交叉引用。
11. [阶段11-API错误触发压缩重试与上下文降级](阶段11-API错误触发压缩重试与上下文降级.md)
    - 解决问题：413、context overflow、long_context_tier 等 API 错误后的 compress 重试与 context 降级。
    - 衔接约束：`max_compression_attempts=3`；每次 compress 后清空 `conversation_history`。
