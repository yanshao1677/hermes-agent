# 阶段6：请求副本归一化与协议清理

## 触发条件

阶段 5（或跳过缓存时阶段 4）完成后，每次 API 请求发送前**无条件**执行。只作用于 `api_messages`，不修改真实历史 `messages`。

## 输入字段

| 字段 | 说明 |
|------|------|
| `api_messages` | 完整请求副本 |

## 判定规则

### R1：协议清理 `_sanitize_api_messages`

与工具容错阶段 1 同函数，在请求副本上执行：

1. 删除 `role` 不在 `{system, user, assistant, tool, function, developer}` 的消息。
2. 删除 orphan `tool` 结果（无 matching assistant tool_call_id）。
3. 为缺失结果的 tool_call 注入 stub：`"[结果不可用——请参阅上面的上下文摘要]"`。

### R2：content 空白归一化

对每条 `api_messages`：若 `content` 为 `str`，执行 `content.strip()`。

### R3：tool_calls 参数 JSON 归一化

对含 `tool_calls` 的 assistant 消息：

```text
for each tc with dict "function":
    args_obj = json.loads(tc["function"]["arguments"])
    tc["function"]["arguments"] = json.dumps(
        args_obj, separators=(",", ":"), sort_keys=True
    )
```

`json.loads` 失败 → **保留**原参数字符串，不中断。

## 状态读写位置

- 写：就地修改 `api_messages`（R1 可能返回新列表再赋回）。
- 不写：`messages`、Session DB。

## 正常路径

1. `api_messages = _sanitize_api_messages(api_messages)`。
2. 对所有 str content 执行 strip。
3. 对所有 tool_calls 尝试 sort_keys 归一化。
4. 将副本交给 `_build_api_kwargs()` 发起请求。

## 分支路径

- 无 tool_calls → 跳过 R3。
- 无效 JSON 参数 → 跳过该 tc 的归一化。
- 未启用 prompt caching → 仍执行 R1–R3（与压缩器是否存在无关）。

## 失败处理

R1/R3 内部捕获异常并降级；不因单条 tool_call 解析失败而中止请求构造。

## 幂等性 / 一致性约束

- **顺序固定**：缓存断点（阶段 5）→ sanitize → strip/sort_keys。注释说明 strip 与 sort_keys 为提升跨回合 prefix cache 命中率；对已打 cache_control 的副本做内容变换，下次请求需重新打 marker。
- 真实历史保留原始空白与 JSON key 顺序；仅 API 副本归一化。

## 代码骨架

```python
def finalize_api_messages(api_messages: list[dict]) -> list[dict]:
    api_messages = sanitize_api_messages(api_messages)
    for am in api_messages:
        if isinstance(am.get("content"), str):
            am["content"] = am["content"].strip()
    for am in api_messages:
        for tc in am.get("tool_calls") or []:
            if isinstance(tc, dict) and "function" in tc:
                try:
                    args = json.loads(tc["function"]["arguments"])
                    tc["function"] = {
                        **tc["function"],
                        "arguments": json.dumps(args, separators=(",", ":"), sort_keys=True),
                    }
                except Exception:
                    pass
    return api_messages
```

## 最小验证清单

1. orphan tool 结果被删除；缺失 stub 被注入。
2. `" content "` strip 为 `"content"`，仅出现在 `api_messages`。
3. `{"b":1,"a":2}` 归一化为 `{"a":2,"b":1}` 紧凑 JSON。
4. 真实历史 `messages` 中 content 与 arguments 不被 strip/sort。
