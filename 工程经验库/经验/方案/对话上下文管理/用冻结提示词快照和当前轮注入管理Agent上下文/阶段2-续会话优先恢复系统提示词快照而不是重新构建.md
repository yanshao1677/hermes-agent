# 阶段2：续会话优先恢复系统提示词快照而不是重新构建

## 触发条件

进入一次对话运行后，真实历史已从传入历史复制完成，并已追加当前用户消息。只有当进程内 `cached_system_prompt` 为空时才进入本阶段；若已存在缓存，直接复用缓存，不读取会话存储，也不重建提示词。

## 输入字段

- `cached_system_prompt`：字符串或空值，进程内会话级系统提示词快照。
- `conversation_history`：消息列表或空值，表示本轮是否带有旧历史。
- `session_db`：可选会话存储，需支持按会话标识读取元数据。
- `session_id`：字符串，用于读取当前会话行。
- `system_message`：可选字符串，仅在需要重建系统提示词时参与构建。

## 判定规则

1. 若 `cached_system_prompt != None`，本阶段结束，输出该缓存。
2. 若 `cached_system_prompt == None`，先令 `stored_prompt = None`。
3. 只有同时满足 `conversation_history` 非空且 `session_db` 存在时，才尝试读取会话元数据。
4. 读取会话行成功且 `row.system_prompt` 为非空字符串时，令 `cached_system_prompt = row.system_prompt`。
5. 上一步未得到非空快照时，调用系统提示词构建器生成新快照，并写入 `cached_system_prompt`。
6. 只有“新建快照”分支才触发会话开始钩子，并尝试把新快照写回会话存储。
7. 读取会话存储失败的动作是跳过恢复并继续走重建分支；写回会话存储失败的动作是记录调试信息并继续主流程。

## 状态读写位置

- 读：进程内 `cached_system_prompt`、会话存储中的 `system_prompt` 字段。
- 写：进程内 `cached_system_prompt`；新建快照后写会话存储的 `system_prompt`。
- 跨进程语义：续会话如果每轮新建进程或新建智能体实例，只有会话存储里的 `system_prompt` 能保持稳定前缀；不能只依赖进程内缓存。

## 正常路径

1. 检查进程内系统提示词缓存。
2. 缓存为空且有旧历史、会话存储时，按 `session_id` 读取会话行。
3. 会话行含非空 `system_prompt` 时，把它提升为当前运行的系统提示词。
4. 若无法恢复，调用系统提示词构建器。
5. 新建系统提示词后，触发一次会话开始钩子。
6. 新建系统提示词后，尝试保存到会话存储。
7. 将最终快照赋给 `active_system_prompt`，供压缩预检和后续 API 请求副本使用。

## 分支路径

- 缓存已存在 → 不读取存储、不构建、不触发会话开始钩子。
- 缓存为空且旧历史存在且存储可用且存储快照非空 → 使用存储快照，不触发新会话钩子，不写回快照。
- 缓存为空但无旧历史 → 构建新快照，触发新会话钩子，尝试写入存储。
- 缓存为空且读取存储异常 → 吞掉读取异常，构建新快照。
- 缓存为空且存储行不存在或 `system_prompt` 为空 → 构建新快照。
- 新建快照写存储失败 → 不阻断主流程。

## 失败处理

- 读取会话存储失败：回退为新建快照。
- 会话开始钩子失败：记录警告，继续。
- 快照写回失败：记录调试信息，继续。

## 幂等性 / 一致性约束

本阶段提供会话级前缀一致性，不提供全局一致性。系统提示词快照一旦恢复，本轮内后续工具循环都必须复用同一个 `active_system_prompt`，不能在工具结果后重新读取记忆或上下文文件重建提示词。压缩阶段是明确例外：压缩后会失效缓存并重建快照。

## 代码骨架

```python
def resolve_system_prompt(state, history, system_message=None):
    if state.cached_system_prompt is not None:
        return state.cached_system_prompt

    stored_prompt = None
    if history and state.session_db:
        try:
            row = state.session_db.get_session(state.session_id)
            if row:
                stored_prompt = row.get("system_prompt") or None
        except Exception:
            stored_prompt = None

    if stored_prompt:
        state.cached_system_prompt = stored_prompt
        return stored_prompt

    prompt = build_system_prompt(system_message)
    state.cached_system_prompt = prompt
    try:
        invoke_session_start_hook(state.session_id)
    except Exception:
        pass
    if state.session_db:
        try:
            state.session_db.update_system_prompt(state.session_id, prompt)
        except Exception:
            pass
    return prompt
```

## 最小验证清单

- `cached_system_prompt` 非空时，不调用 `session_db.get_session()`。
- 传入旧历史且存储中存在 `system_prompt` 时，不调用系统提示词构建器。
- 存储读取抛异常时，仍返回新构建的提示词。
- 新建提示词后调用一次存储写入；恢复旧提示词时不重复写入。
- 压缩阶段之外，同一轮多次 API 调用使用同一个 `active_system_prompt`。
