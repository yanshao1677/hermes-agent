# 阶段1：当前轮临时上下文只注入用户消息副本

## 触发条件

每次准备发起模型 API 请求时，在从真实历史复制请求消息的过程中触发。前提：本轮 `current_turn_user_idx` 已确定，且主循环前已收集 `_ext_prefetch_cache` 与 `_plugin_user_context`（允许为空串）。

## 输入字段

| 字段名 | 类型 | 是否可选 | 来源 |
|--------|------|----------|------|
| `messages` | `List[Dict]` | 必选 | 真实历史 |
| `current_turn_user_idx` | `int` | 必选 | 当前轮 user 消息下标 |
| `_ext_prefetch_cache` | `str` | 可选，默认 `""` | 阶段 3 收集的外部记忆 prefetch |
| `_plugin_user_context` | `str` | 可选，默认 `""` | 阶段 3 收集的插件上下文 |

## 判定规则

### R1：副本创建

对 `messages` 中第 `idx` 条消息执行 `api_msg = msg.copy()`（浅拷贝）。

### R2：注入条件

仅当 `idx == current_turn_user_idx` 且 `msg.get("role") == "user"` 时允许追加临时上下文。

### R3：注入顺序与格式

```text
_injections = []
if _ext_prefetch_cache not empty:
    _fenced = build_memory_context_block(_ext_prefetch_cache)  # 非空则包 <memory-context>
    if _fenced not empty: append _fenced
if _plugin_user_context not empty:
    append _plugin_user_context
if _injections not empty AND api_msg["content"] is str:
    api_msg["content"] = base + "\n\n" + join(_injections, "\n\n")
```

### R4：assistant 消息字段映射

若 `role == assistant` 且存在 `reasoning`：复制到 `reasoning_content`；随后从 `api_msg` 移除 `reasoning`、`finish_reason`、`_thinking_prefill`。

### R5：严格 API 工具调用字段

若启用 strict API 模式：对 assistant 消息调用 `_sanitize_tool_calls_for_strict_api(api_msg)`，剥离 Codex 扩展字段（详见工具容错阶段 9）。

## 状态读写位置

- 读：`messages`、本轮缓存的 prefetch/插件字符串。
- 写：`api_messages` 列表（进程内，不持久化）。
- 不写：真实历史、会话 DB、系统提示词快照。

## 正常路径

1. 遍历 `messages`，逐条浅拷贝。
2. 命中当前轮 user 时按 R3 追加临时上下文。
3. 对 assistant 消息执行 R4、R5。
4. 将 `api_msg` 追加到 `api_messages`。
5. 交给阶段 4 插入 system/prefill。

## 分支路径

- `_injections` 为空 → user 副本内容不变，继续。
- user 的 `content` 非字符串 → 不追加注入（跳过 R3 拼接）。
- prefetch/插件为空 → 对应项跳过。

## 失败处理

本阶段不发起 IO。注入源在阶段 3 已容错为空串；`build_memory_context_block("")` 返回空串，不抛错。

## 幂等性 / 一致性约束

- 每次 API 调用必须从真实历史**重新**复制并注入，不能在上次 `api_messages` 上叠加。
- 同一用户轮次内多次工具循环复用**同一份** prefetch/插件缓存，但每次都重新执行本阶段复制逻辑。
- 真实历史中的 user 消息内容不得被本阶段修改。

## 代码骨架

```python
def inject_turn_context_into_user_copy(
    messages: list[dict],
    current_turn_user_idx: int,
    ext_prefetch_cache: str = "",
    plugin_user_context: str = "",
) -> list[dict]:
    api_messages = []
    for idx, msg in enumerate(messages):
        api_msg = msg.copy()
        if idx == current_turn_user_idx and msg.get("role") == "user":
            injections = []
            if ext_prefetch_cache:
                fenced = build_memory_context_block(ext_prefetch_cache)
                if fenced:
                    injections.append(fenced)
            if plugin_user_context:
                injections.append(plugin_user_context)
            base = api_msg.get("content", "")
            if injections and isinstance(base, str):
                api_msg["content"] = base + "\n\n" + "\n\n".join(injections)
        if msg.get("role") == "assistant" and msg.get("reasoning"):
            api_msg["reasoning_content"] = msg["reasoning"]
        for field in ("reasoning", "finish_reason", "_thinking_prefill"):
            api_msg.pop(field, None)
        api_messages.append(api_msg)
    return api_messages
```

## 最小验证清单

1. 注入后 `messages[current_turn_user_idx]["content"]` 与注入前相同。
2. 历史中的其他 user 消息不被追加临时上下文。
3. 同一轮第 2 次 API 调用仍能看到 prefetch，且 `messages` 长度只增加 assistant/tool 条目。
4. prefetch 用 `<memory-context>` 围栏包裹；插件上下文原样追加在 prefetch 之后。
5. `reasoning` 只出现在 `api_messages` 的 `reasoning_content`，不出现在真实历史。
