# 阶段4：系统消息拼接与预填充消息插入请求副本

## 触发条件

阶段 1 已生成不含 system 的历史消息副本 `api_messages`，且 `active_system_prompt`（稳定快照）已就绪。

## 输入字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `api_messages` | `List[Dict]` | 阶段 1 输出的历史副本 |
| `active_system_prompt` | `str` | 稳定系统提示词快照 |
| `ephemeral_system_prompt` | `str` | 可选，调用级临时 system 补充 |
| `prefill_messages` | `List[Dict]` | 可选，少样本 priming，不持久化 |

## 判定规则

### R1：effective_system 拼接

```text
effective_system = active_system_prompt or ""
if ephemeral_system_prompt not empty:
    effective_system = (effective_system + "\n\n" + ephemeral_system_prompt).strip()
```

插件 `pre_llm_call` 上下文**不得**拼入 system；只能进 user 副本（阶段 1）。

### R2：system 消息插入

若 `effective_system` 非空：

```text
api_messages = [{"role": "system", "content": effective_system}] + api_messages
```

### R3：prefill 插入位置

```text
sys_offset = 1 if effective_system else 0
for idx, pfm in enumerate(prefill_messages):
    api_messages.insert(sys_offset + idx, pfm.copy())
```

最终顺序：`[system?] + [prefill...] + [历史副本...]`。

## 状态读写位置

- 写：`api_messages`（请求副本）。
- 不写：真实历史、会话 DB、`cached_system_prompt`（ephemeral 不刷新快照）。

## 正常路径

1. 拼接 `effective_system`。
2. 非空则插入 system 消息到列表头部。
3. 若有 `prefill_messages`，按 R3 插入 system 之后、历史之前。
4. 交给阶段 5（缓存）与阶段 6（归一化）。

## 分支路径

- 无稳定快照且无 ephemeral → 不插入 system；prefill 从 index 0 开始。
- `prefill_messages` 为空 → 跳过插入。
- `ephemeral_system_prompt` 为空 → `effective_system` 等于稳定快照。

## 失败处理

本阶段无外部 IO。prefill 仅 `copy()` 插入，不做 schema 校验；不兼容字段依赖阶段 6 清理或 API 报错。

## 幂等性 / 一致性约束

- 每次 API 调用必须重新执行 R1–R3，不能保留上次的 system/prefill 层。
- prefill 与 ephemeral system 永不写入真实历史或 Session DB。

## 代码骨架

```python
def prepend_system_and_prefill(
    api_messages: list[dict],
    active_system_prompt: str,
    ephemeral_system_prompt: str = "",
    prefill_messages: list[dict] | None = None,
) -> list[dict]:
    effective_system = active_system_prompt or ""
    if ephemeral_system_prompt:
        effective_system = (effective_system + "\n\n" + ephemeral_system_prompt).strip()
    if effective_system:
        api_messages = [{"role": "system", "content": effective_system}] + api_messages
    offset = 1 if effective_system else 0
    for i, pfm in enumerate(prefill_messages or []):
        api_messages.insert(offset + i, dict(pfm))
    return api_messages
```

## 最小验证清单

1. 有 system + 2 条 prefill 时，顺序为 system → prefill1 → prefill2 → 历史。
2. 无 system 时 prefill 从第 0 条开始。
3. `messages` 原列表不含 system 与 prefill。
4. 同一轮多次 API 调用中 `effective_system` 不变（ephemeral 不变时）。
