# 用受控上下文注入和受限工具集实现子智能体隔离

## 1. 问题

复杂任务常需要拆给子智能体并行探索，但如果子智能体继承完整父历史和全部工具，会带来三类风险：主上下文被子任务细节污染；子任务写共享记忆或跨平台发送消息；子智能体递归委托导致失控并发。

## 2. 适用约束

- 父智能体需要把若干相互独立的子问题交给子智能体。
- 子智能体可使用文件、终端、网络等工具，但不应获得比父级更多的能力。
- 父上下文只需要子任务最终摘要，不需要子任务每一步工具轨迹。

## 3. 核心思路

子智能体用全新会话运行，只注入任务目标、显式上下文和可验证工作目录提示；工具集取父级可用工具集交集并剥离危险工具；用深度上限与并发上限防止递归和风暴。

## 4. 通用结构

- **DelegationRequest**：单任务目标或任务列表。
- **ChildPromptBuilder**：把目标、显式上下文、工作目录提示拼成子智能体临时系统提示。
- **ToolsetRestrictor**：计算子工具集，剥离委托、交互、共享记忆、跨平台消息和代码执行等工具。
- **DepthGuard**：限制委托深度。
- **ConcurrencyGuard**：限制同批子任务数量。
- **ResultReducer**：父智能体只接收子任务摘要和状态。

## 5. 处理流程

1. 校验调用方必须提供父智能体上下文。
2. 检查当前委托深度，达到上限则拒绝。
3. 解析任务列表；列表长度不得超过最大并发子任务数。
4. 为每个子任务构造聚焦系统提示，只包含目标、显式上下文和工作目录提示。
5. 计算子工具集：若请求指定工具集，取其与父级工具集交集；未指定则从父级或默认工具集派生；再剥离禁用工具集。
6. 创建子智能体时跳过父级上下文文件和记忆快照，并传入父会话标识作为子会话父标识。
7. 子智能体独立运行；父级等待完成并收集摘要。
8. 批量模式用线程池并发执行，但并发数不超过配置上限。

## 6. 异常处理

- 无父智能体上下文：返回工具错误。
- 深度超限：返回 JSON 错误，不创建子智能体。
- 任务数超限：返回工具错误，要求减少任务数或调整配置。
- 子任务执行异常：作为子任务结果记录，不把异常抛穿父任务。

## 7. 具体语言实现

```python
BLOCKED = {"delegate_task", "clarify", "memory", "send_message", "execute_code"}
MAX_DEPTH = 2
DEFAULT_MAX_CHILDREN = 3

def build_child(goal, context, requested_toolsets, parent):
    if parent.delegate_depth >= MAX_DEPTH:
        return {"error": "delegation depth limit reached"}
    parent_toolsets = set(parent.enabled_toolsets)
    if requested_toolsets:
        child_toolsets = [t for t in requested_toolsets if t in parent_toolsets]
    else:
        child_toolsets = list(parent_toolsets)
    child_toolsets = strip_toolsets_containing_only_blocked_tools(child_toolsets, BLOCKED)
    prompt = f"YOUR TASK:\n{goal}\n\nCONTEXT:\n{context or ''}"
    return Agent(
        toolsets=child_toolsets,
        ephemeral_system_prompt=prompt,
        skip_memory=True,
        skip_context_files=True,
        parent_session_id=parent.session_id,
    )
```

## 8. 测试点

- 子智能体没有父历史，只接收目标和显式上下文。
- 子工具集不能包含父级没有的工具。
- 子工具集剥离递归委托、共享记忆和跨平台消息工具。
- 深度达到上限时拒绝。
- 批量任务数超过上限时拒绝。

## 9. 适用场景 / 不适用场景

适合代码库探索、资料收集、相互独立的分析任务。不适合需要共享同一事务状态、强一致锁或连续人工澄清的流程。

## 10. 风险与反模式

- 把完整父历史塞给子智能体会抵消隔离收益。
- 允许子智能体写共享记忆会污染长期偏好。
- 允许递归委托会导致成本和并发不可控。
