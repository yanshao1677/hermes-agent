# 阶段2：子智能体工具集取父级交集并剥离禁用工具

## 触发条件

创建子智能体前，需要确定子智能体可用工具集。

## 输入字段

- `toolsets`：委托请求指定的工具集列表，可选。
- `parent_enabled_toolsets`：父智能体启用工具集。
- 默认工具集列表。
- 禁用工具名集合和应剥离工具集名集合。

## 判定规则

1. 若请求显式指定工具集，子工具集取“请求工具集 ∩ 父级工具集”。
2. 若请求未指定但父级有工具集，子工具集从父级工具集派生。
3. 若父级工具集不可用，子工具集使用默认工具集。
4. 剥离只包含禁用工具或明确禁止的工具集。
5. 禁用工具至少包括递归委托、用户澄清、共享记忆、跨平台消息和代码执行。
6. 子智能体创建时同时设置 `skip_context_files=True` 和 `skip_memory=True`。

## 状态读写位置

- 读：父智能体工具集配置。
- 写：子智能体构造参数中的工具集、跳过记忆和上下文文件标志。

## 正常路径

1. 读取父级工具集。
2. 根据请求参数计算初始子工具集。
3. 调用剥离函数删除禁止工具集。
4. 用结果创建子智能体。
5. 子智能体启动时不会加载父级记忆快照和上下文文件。

## 分支路径

- 请求工具集包含父级没有的工具集 → 丢弃该项。
- 请求工具集全部被剥离 → 证据不足：最终空工具集时行为需实现侧确认。
- 父级无工具集 → 使用默认工具集后再剥离。

## 失败处理

证据不足：工具集名无效时是否返回错误或静默忽略，需结合委托参数校验继续深读。本卡确认权限提升被父级交集规则阻断。

## 幂等性 / 一致性约束

子智能体不能获得父智能体没有的工具能力；剥离规则必须在交集之后执行，避免请求用别名绕开禁用工具。

## 代码骨架

```python
BLOCKED_TOOLSETS = {"delegation", "clarify", "memory", "code_execution"}

def resolve_child_toolsets(requested, parent_toolsets, defaults):
    base = [t for t in requested if t in parent_toolsets] if requested else list(parent_toolsets or defaults)
    return [t for t in base if t not in BLOCKED_TOOLSETS]
```

## 最小验证清单

- 请求父级没有的工具集时，子工具集不包含它。
- 请求 `memory` 工具集时，子工具集不包含它。
- 未请求工具集时，从父级工具集派生。
- 子智能体构造参数中 `skip_memory=True`。
