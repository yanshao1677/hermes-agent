# 阶段1：子任务只接收目标和显式上下文不继承父历史

## 触发条件

父智能体调用子任务委托工具，并提供单个目标或任务列表。

## 输入字段

- `goal`：子任务目标，必需。
- `context`：显式传给子任务的上下文，可选。
- `workspace_path`：从父运行环境解析出的本地工作目录提示，可选。

## 判定规则

1. 子系统提示必须包含 `YOUR TASK` 段，内容为目标。
2. 只有 `context` 非空时才追加 `CONTEXT` 段。
3. 只有能解析出绝对且存在的目录时，才追加工作目录提示。
4. 子智能体不接收父会话历史列表。
5. 父智能体上下文只记录委托调用和子任务摘要，不记录子任务中间工具轨迹。

## 状态读写位置

- 写：子智能体的临时系统提示。
- 不写：父真实历史中的子任务中间消息。

## 正常路径

1. 从委托请求读取目标和上下文。
2. 尝试解析工作目录提示。
3. 构造子系统提示。
4. 创建子智能体并运行全新对话。
5. 子任务结束后，把摘要结果返回父级。

## 分支路径

- 上下文为空 → 省略 `CONTEXT` 段。
- 工作目录无法确认 → 不注入路径，避免教子智能体假路径。

## 失败处理

证据不足：目标为空时的完整校验分支未在本卡深读；方案层确认委托工具会规范化任务列表并返回工具错误。

## 幂等性 / 一致性约束

子任务上下文是显式传递语义，不是父历史继承语义。需要子任务知道的信息必须由父级放进 `context` 或目标中。

## 代码骨架

```python
def build_child_prompt(goal, context=None, workspace_path=None):
    parts = ["You are a focused subagent working on a specific delegated task.", "", f"YOUR TASK:\n{goal}"]
    if context and context.strip():
        parts.append(f"\nCONTEXT:\n{context}")
    if workspace_path and is_existing_absolute_dir(workspace_path):
        parts.append(f"\nWORKSPACE PATH:\n{workspace_path}")
    return "\n".join(parts)
```

## 最小验证清单

- 子提示包含目标。
- 空上下文时不出现 `CONTEXT` 段。
- 无有效目录时不注入路径。
- 子智能体运行时不传父历史。
