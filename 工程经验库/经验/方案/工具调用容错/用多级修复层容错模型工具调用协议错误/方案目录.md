# 方案目录：用多级修复层容错模型工具调用协议错误

> 架构总纲见《用三层修复层容错模型工具调用协议错误》。

## 所属方案

- [用多级修复层容错模型工具调用协议错误](../用多级修复层容错模型工具调用协议错误.md)

## 原子卡（按执行顺序）

### 请求前层

1. [阶段1-请求前清理孤立工具结果并补齐缺失结果](阶段1-请求前清理孤立工具结果并补齐缺失结果.md) —— API 副本：角色白名单、删孤立 tool、补 stub。
2. [阶段2-Unicode代理字符清理避免JSON序列化失败](阶段2-Unicode代理字符清理避免JSON序列化失败.md) —— 用户输入与 API 重试路径上的 `\ud800-\udfff` 替换。
9. [阶段9-严格API剥离tool_calls扩展字段](阶段9-严格API剥离tool_calls扩展字段.md) —— 非 codex 模式去掉 `call_id`/`response_item_id`（与阶段 1 同处于 API 构造链）。

### 响应后层

3. [阶段3-工具名模糊修复后再校验](阶段3-工具名模糊修复后再校验.md) —— 小写/规范化/difflib 0.7。
4. [阶段4-无效工具名注入tool错误并限制重试](阶段4-无效工具名注入tool错误并限制重试.md) —— **tool role** 错误；3 次后 partial。
5. [阶段5-工具参数JSON校验与空参归一化](阶段5-工具参数JSON校验与空参归一化.md) —— dict/list 序列化、空白→`{}`、非 string→str。
6. [阶段6-截断工具参数拒绝执行与流式finish_reason联动](阶段6-截断工具参数拒绝执行与流式finish_reason联动.md) —— 非 `}`/`]` 结尾拒绝；`truncated_tool_call_retries≤1`。

### 执行前层

7. [阶段7-委托调用截断与同批去重](阶段7-委托调用截断与同批去重.md) —— max_concurrent_children 默认 3；`(name,args)` 去重。
8. [阶段8-安全并发判定与线程池执行](阶段8-安全并发判定与线程池执行.md) —— 白名单/路径重叠/clarify 禁止；max_workers≤8。

## 阶段衔接约束

- 阶段 1 → 9 在每次 API 请求构造链末尾（1 在 7804，9 在 strict API 复制时）。
- 阶段 3 → 4 → 5 → 6 顺序固定；6 通过后才进入 7 → 8。
- 阶段 4/5 的重试计数在 `run_conversation` 每 user turn 开头归零（7451-7452）。
