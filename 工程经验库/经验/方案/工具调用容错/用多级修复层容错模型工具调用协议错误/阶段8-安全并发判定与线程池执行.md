# 阶段8：安全并发判定与线程池执行

## 触发条件

阶段 7 后 `_execute_tool_calls()`；根据 `_should_parallelize_tool_batch(tool_calls)` 选并发或串行。

## 输入字段

| 常量 | 成员 |
|------|------|
| `_NEVER_PARALLEL_TOOLS` | `clarify` |
| `_PARALLEL_SAFE_TOOLS` | `ha_get_state`, `ha_list_entities`, `ha_list_services`, `read_file`, `search_files`, `session_search`, `skill_view`, `skills_list`, `vision_analyze`, `web_extract`, `web_search` |
| `_PATH_SCOPED_TOOLS` | `read_file`, `write_file`, `patch` |
| `_MAX_TOOL_WORKERS` | **8** |

## 判定规则（`_should_parallelize_tool_batch`）

1. `len(tool_calls) <= 1` → False。
2. 任一工具名 ∈ `_NEVER_PARALLEL_TOOLS` → False。
3. 对每个 tool_call：
   - `json.loads(arguments)` 失败或非 dict → False。
   - 若 ∈ `_PATH_SCOPED_TOOLS`：提取 `path` 参数规范化路径；缺 path → False；与已保留路径 `_paths_overlap`（公共前缀相等）→ False；否则加入 reserved。
   - 若 ∉ `_PATH_SCOPED_TOOLS` 且 ∉ `_PARALLEL_SAFE_TOOLS` → False。
4. 全部通过 → True。

**并发执行**

- `ThreadPoolExecutor(max_workers=min(num_tools, 8))`
- 每线程 `_invoke_tool`；异常 → `执行工具 'X' 时出错: ...`
- `results[i]` 按原索引；None → `线程未返回结果` 错误串
- 按 **原 tool_calls 顺序** append tool messages

**串行执行**

- 每工具前检查 `_interrupt_requested` → 剩余 tc  append `[工具执行已取消 — {name} 因用户中断被跳过]`
- JSON parse 失败兜底 `{}`（6887-6893）

## 状态读写位置

- 写：messages 追加 tool 结果。
- 读：tool_calls 列表（已 dedupe）。

## 正常路径

2 个 web_search → 并发 → 顺序写回。

## 分支路径

- write_file 两路径重叠 → 串行。
- terminal + read_file → 串行（terminal 不在 safe 集）。
- clarify 在批 → 串行。

## 失败处理

单工具异常不阻断批次；interrupt 跳过未启动工具。

## 幂等性 / 一致性约束

结果顺序必须与原 tool_call_id 顺序一致。并发不改变 messages 中 tool 消息相对顺序。

## 代码骨架

```python
NEVER = frozenset({"clarify"})
SAFE = frozenset({"read_file", "web_search", ...})  # 见上文完整列表
PATH_SCOPED = frozenset({"read_file", "write_file", "patch"})

def should_parallelize(tool_calls):
    if len(tool_calls) <= 1:
        return False
    if any(tc.function.name in NEVER for tc in tool_calls):
        return False
    reserved = []
    for tc in tool_calls:
        args = json.loads(tc.function.arguments)  # must succeed
        if tc.function.name in PATH_SCOPED:
            p = scope_path(args.get("path"))
            if p is None or any(overlap(p, r) for r in reserved):
                return False
            reserved.append(p)
        elif tc.function.name not in SAFE:
            return False
    return True
```

## 最小验证清单

- 单工具不并发。
- clarify 批次串行。
- 两 read_file 不同 path 可并发；相同 path 前缀重叠则串行。
- 并发后 tool 消息顺序与 tool_calls 一致。
