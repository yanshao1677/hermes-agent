# 阶段1：请求前清理孤立工具结果并补齐缺失结果

## 触发条件

每次构造完 `api_messages`、发起 LLM 请求之前；无条件执行（不依赖是否启用上下文压缩）。

## 输入字段

- `messages` / `api_messages`：即将发送给 API 的列表副本。
- 助手消息 `tool_calls[].id`（dict 或 object，经 `_get_tool_call_id_static` 提取）。
- 工具消息 `tool_call_id`。

## 判定规则

1. **角色白名单**：`role ∉ {system,user,assistant,tool,function,developer}` → 删除该消息。
2. 收集所有助手消息中非空 `tool_call_id` → `surviving_call_ids`。
3. 收集所有 tool 消息中非空 `tool_call_id` → `result_call_ids`。
4. `orphaned_results = result_call_ids - surviving_call_ids` → 删除对应 tool 消息。
5. `missing_results = surviving_call_ids - result_call_ids` → 在每个含缺失 id 的 assistant 消息**之后**插入 stub：
   - `role: tool`
   - `tool_call_id: cid`
   - `content: "[结果不可用——请参阅上面的上下文摘要]"`
6. 无 id 的 tool_call / tool 结果不参与集合运算。

## 状态读写位置

- 只写 API 请求副本（`run_agent.py:7804` 调用点）。
- 不修改真实 `messages`、不写 session DB。

## 正常路径

过滤角色 → 收集 id → 删 orphan → 补 stub → 返回。

## 分支路径

- 无 orphan 且无 missing → 列表不变（或仅角色过滤后等价）。
- `tool_calls` 非 list → 当作 `or []`。

## 失败处理

纯内存操作，无 I/O 异常分支。畸形 tool_call 元素缺 id 则跳过配对。

## 幂等性 / 一致性约束

对同一 api_messages 连续两次 sanitize：第二次不应再增删 stub。stub 内容非真实执行结果，仅满足 API 协议。

## 代码骨架

```python
VALID = frozenset({"system", "user", "assistant", "tool", "function", "developer"})
STUB = "[结果不可用——请参阅上面的上下文摘要]"

def sanitize_api_messages(messages):
    messages = [m for m in messages if m.get("role") in VALID]
    call_ids = {cid for m in messages if m.get("role") == "assistant"
                for tc in (m.get("tool_calls") or [])
                if (cid := get_id(tc))}
    result_ids = {m.get("tool_call_id") for m in messages
                  if m.get("role") == "tool" and m.get("tool_call_id")}
    messages = [m for m in messages
                if not (m.get("role") == "tool" and m.get("tool_call_id") in result_ids - call_ids)]
    missing = call_ids - result_ids
    if not missing:
        return messages
    out = []
    for m in messages:
        out.append(m)
        if m.get("role") == "assistant":
            for tc in m.get("tool_calls") or []:
                if (cid := get_id(tc)) in missing:
                    out.append({"role": "tool", "tool_call_id": cid, "content": STUB})
    return out
```

## 最小验证清单

- 无效 role 消息被删。
- orphan tool 结果被删。
- 有 assistant tool_calls 无 tool 结果时补一条 stub。
- 二次 sanitize 不重复 stub。
