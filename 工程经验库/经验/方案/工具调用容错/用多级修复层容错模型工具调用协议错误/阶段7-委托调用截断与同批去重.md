# 阶段7：委托调用截断与同批去重

## 触发条件

阶段 5–6 通过后，append assistant 消息**之前**，就地修改 `assistant_message.tool_calls`。

## 输入字段

| 字段 | 默认 |
|------|------|
| `max_concurrent_children` | config `delegation.max_concurrent_children` → env → **3** |
| 去重键 | `(tc.function.name, tc.function.arguments)` 字符串元组 |

## 判定规则

**`_cap_delegate_task_calls(tool_calls)`**

1. 统计 `name == "delegate_task"` 数量。
2. 若 `delegate_count <= max_children` → 原样返回。
3. 否则遍历：保留前 `max_children` 个 delegate_task；**非 delegate 全部保留**；多余 delegate 丢弃；warning 日志。

**`_deduplicate_tool_calls(tool_calls)`**

1. 按顺序遍历，键 `(name, arguments)`。
2. 首次出现保留；重复键跳过并 warning。
3. 若无重复，返回**原 list 对象**（优化）；否则新 list。

**顺序**：先 cap delegate，再 deduplicate（`9370-9375`）。

## 状态读写位置

- 写：当前 assistant_message 的 tool_calls 列表（内存）。
- 随后 `_build_assistant_message` 写入 messages 的是已截断/去重版本。

## 正常路径

2 个 delegate + 1 个 read_file → 全部保留。

## 分支路径

- 5 个 delegate，max=3 → 后 2 个 delegate 丢弃，read_file 仍在。
- 两个完全相同 `(terminal, same_json)` → 只执行第一个。

## 失败处理

无异常；截断/去重是 lossy 但 deterministic。

## 幂等性 / 一致性约束

去重仅同一 assistant 消息批次内，不跨 API 迭代。delegate cap 与 delegate_tool 内部 tasks 列表限制互补（本层限制**多个独立 delegate_task 调用**）。

## 代码骨架

```python
def cap_delegate(calls, max_children=3):
    kept = 0
    out = []
    for tc in calls:
        if tc.function.name == "delegate_task":
            if kept < max_children:
                out.append(tc)
                kept += 1
        else:
            out.append(tc)
    return out

def dedupe(calls):
    seen, out = set(), []
    for tc in calls:
        key = (tc.function.name, tc.function.arguments)
        if key not in seen:
            seen.add(key)
            out.append(tc)
    return out if len(out) < len(calls) else calls
```

## 最小验证清单

- 同批 5 个 delegate_task 只留 3 个。
- 非 delegate 不被 cap 误删。
- 同名同参两次只保留第一次。
