# 阶段1：注册内置提供者与单一外部提供者

## 触发条件

Agent 初始化：`memory.provider` 配置非空且 `load_memory_provider(name)` 返回可用实例；`skip_memory=False`。

## 输入字段

| 字段 | 说明 |
|------|------|
| `memory.provider` | 字符串，如 honcho / mem0 |
| `_mp.is_available()` | 凭证/依赖检查 |
| `session_id`, `platform`, `hermes_home`, `user_id` | 传入 `initialize_all(**kwargs)` |

## 判定规则

1. `MemoryManager()` 空列表初始化；`_has_external=False`。
2. `add_provider(provider)`：
   - `provider.name == "builtin"` → 始终 append。
   - 非 builtin：若 `_has_external` 已为 True → `logger.warning` + **return 不添加**。
   - 否则 append 且 `_has_external=True`。
3. 工具索引：遍历 `provider.get_tool_schemas()`，`tool_name` 首次写入 `_tool_to_provider`；冲突则 warning 忽略后者。
4. 若 `providers` 非空：`initialize_all(session_id=..., platform=..., hermes_home=..., agent_context="primary", user_id=可选)`。
5. 若注册后无 provider 或 init 失败：`self._memory_manager = None`。
6. 成功时：`get_all_tool_schemas()` 合并进 agent `tools` 列表。

## 状态读写位置

- 写：`_providers`、`_tool_to_provider`、`_has_external`。
- provider 内部状态：由 `initialize()` 建立。

## 正常路径

1. 读 config provider 名。
2. load plugin → `add_provider`。
3. `initialize_all`。
4. 工具 schema 进入模型可见工具集。

## 分支路径

- provider 不可用 → 不创建 manager。
- 第二个 external → 拒绝注册。
- init 单 provider 异常 → warning，其它 provider 仍可用。

## 失败处理

初始化外层 try/except → `_memory_manager=None`，本地 MemoryStore 仍可用。

## 幂等性 / 一致性约束

运行时不可热插拔第二个 external；改 provider 需重启 agent。builtin 与 external 工具名不得冲突。

## 代码骨架

```python
class MemoryManager:
    def add_provider(self, provider):
        if provider.name != "builtin" and self._has_external:
            log.warning("reject second external %s", provider.name)
            return
        if provider.name != "builtin":
            self._has_external = True
        self._providers.append(provider)
        for schema in provider.get_tool_schemas():
            name = schema.get("name")
            if name and name not in self._tool_to_provider:
                self._tool_to_provider[name] = provider
```

## 最小验证清单

- 连续 add 两个 external 时第二个不在 `_providers`。
- builtin + 一个 external 共存。
- 冲突 tool name 只保留先注册 provider 的路由。
