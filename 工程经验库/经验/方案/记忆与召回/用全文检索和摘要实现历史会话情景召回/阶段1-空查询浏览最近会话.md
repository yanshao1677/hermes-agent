# 阶段1：空查询浏览最近会话

## 触发条件

`session_search(query=...)` 且 `not query or not query.strip()`。

## 输入字段

| 字段 | 默认 | 说明 |
|------|------|------|
| `query` | — | 空或纯空白 |
| `limit` | 3 | 入口 `min(limit, 5)` |
| `current_session_id` | None | 用于排除当前 lineage |
| `db` | 必填 | SessionStore |

## 判定规则

1. 调用 `_list_recent_sessions(db, limit, current_session_id)`。
2. `list_sessions_rich(limit=limit+5, exclude_sources=["tool"])` — 多取 5 条以便跳过 current 后仍够。
3. 解析 current lineage root：沿 `parent_session_id` 向上，visited 防环；失败则 fallback `current_session_id`。
4. 遍历 sessions：
   - skip：`sid == current_root` 或 `sid == current_session_id`。
   - skip：`parent_session_id` 非空（子/委托会话不直接展示）。
5. 收集字段：`session_id`, `title`, `source`, `started_at`, `last_active`, `message_count`, `preview`。
6. 达到 `limit` 停止。
7. 返回 JSON：`success=true`, `mode="recent"`, `results`, `count`, 提示用语 keyword 搜索。

**不调 LLM、不做 FTS。**

## 状态读写位置

- 只读 DB sessions 表。
- 不写。

## 正常路径

1. 空 query 进入。
2. 拉 recent 列表。
3. 过滤 current 与 child sessions。
4. 返回 metadata JSON。

## 分支路径

- DB 异常 → `tool_error("Failed to list recent sessions: ...")`。
- 过滤后无结果 → `count=0`, `results=[]` 仍 success。

## 失败处理

except 记录 error 日志并返回 tool_error JSON。

## 幂等性 / 一致性约束

只读；preview 字段来自 DB 缓存，非实时全文。

## 代码骨架

```python
def session_search(query, limit=3, db=None, current_session_id=None):
    limit = min(limit, 5)
    if not query or not query.strip():
        return list_recent_sessions(db, limit, current_session_id)
    ...
```

## 最小验证清单

- `query=""` 返回 `mode=recent` 且无 auxiliary LLM 调用。
- 当前 session 不在 results。
- 有 `parent_session_id` 的 session 被跳过。
