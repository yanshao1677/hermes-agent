# 阶段3：命中会话并发摘要失败时降级 preview

## 触发条件

阶段 2 产出非空 `seen_sessions` 候选 map。

## 输入字段

| 字段 | 值 |
|------|-----|
| `MAX_SESSION_CHARS` | 100_000 |
| `MAX_SUMMARY_TOKENS` | 10000 |
| 单会话重试 | 最多 3 次 |
| 并发整体超时 | 60 秒 |
| raw preview 长度 | 500 字符 + `\n…[truncated]` |

## 判定规则

1. 每候选：`load_messages(session_id)` → `_format_conversation()`。
2. TOOL 消息 content >500 → 保留前 250 + 标记 + 后 250。
3. `_truncate_around_matches(text, query, max_chars=100_000)`：无 query term 命中则从头取窗口。
4. 并发提交各 `_summarize_session(text, query)`：auxiliary LLM；不可用时返回 None。
5. 单会话最多 3 次 retry；仍失败 → None。
6. `asyncio.wait_for(gather, timeout=60)` — 超时 → 返回错误 JSON 建议缩小 query/limit。
7. 对每个结果：summary 非空 → 用 summary；否则：
   ```
   "[Raw preview — summarization unavailable]\n" + text[:500] + ("\n…[truncated]" if text else "")
   ```
8. 加载单会话失败 → log error；证据不足：是否仍返回 partial results 取决于外层 loop。

## 状态读写位置

- 读：session messages。
- 写：工具返回 JSON only。

## 正常路径

1. 格式化 + 截断。
2. 并发摘要。
3. 组装 per-session 结果项（metadata + summary 字段）。

## 分支路径

- 摘要模型 RuntimeError → None → preview。
- 整体超时 → 不 partial return，整 tool 错误。

## 失败处理

preview 降级保证命中会话仍可见。超时为硬失败。

## 幂等性 / 一致性约束

摘要非 deterministic；preview 非 verbatim 证据。

## 代码骨架

```python
async def summarize_all(tasks, timeout=60):
    return await asyncio.wait_for(asyncio.gather(*tasks), timeout=timeout)

def fallback_preview(text):
    prefix = "[Raw preview — summarization unavailable]\n"
    body = (text[:500] + "\n…[truncated]") if text else "No preview available."
    return prefix + body
```

## 最小验证清单

- 文本 200k 字符截断后 ≤100k。
- 摘要返回 None 时 summary 字段含 Raw preview。
- 60s 超时返回 error JSON。
