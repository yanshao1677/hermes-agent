# 阶段2：记忆工具写入时更新实时状态但不刷新快照

## 触发条件

模型调用 `memory` 工具，且 `store` 非空；`action` 为 `add`、`replace` 或 `remove`；`target` 为 `memory` 或 `user`。

## 输入字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | enum | 是 | `add` / `replace` / `remove` |
| `target` | enum | 是 | `memory` / `user` |
| `content` | string | add/replace 必填 | 写入内容，会先 `strip()` |
| `old_text` | string | replace/remove 必填 | 子串匹配定位条目 |
| `store` | MemoryStore | 是 | 运行时注入 |

## 判定规则

**公共前置（三种 action）**

1. `target` 非法 → 返回 `success: false`。
2. 写路径：`with _file_lock(path)` → `_reload_target(target)` 从磁盘重读（跨会话并发安全）。
3. 写盘：`save_to_disk()` = temp 文件 + `fsync` + `os.replace` 原子替换。

**add**

1. `content.strip()` 为空 → 拒绝。
2. `_scan_memory_content(content)` 非空 → 拒绝，返回 blocked 原因。
3. `content in entries` → 返回 success + message「已存在」，不追加。
4. `len(DELIMITER.join(entries + [content])) > char_limit` → 拒绝，返回 current usage。
5. 否则 append → `_set_entries` → `save_to_disk`。

**replace**

1. `old_text`、`new_content` 非空；扫描 `new_content`。
2. `matches = [(i,e) for i,e in enumerate(entries) if old_text in e]`。
3. `len(matches)==0` → 拒绝；`len(matches)>1` 且 unique 文本 >1 → 拒绝并返回 previews（前 80 字符）。
4. identical duplicates → 只替换 `matches[0]`。
5. 替换后总 char 超 limit → 拒绝。

**remove**

1. 同 replace 的 match 规则；`entries.pop(idx)` → save。

**快照**

- 任何成功写入**不**修改 `_system_prompt_snapshot`。

## 状态读写位置

- 读：磁盘（锁内 reload）、内存 entries。
- 写：内存 entries、磁盘文件。
- 不写：`_system_prompt_snapshot`。

## 正常路径

1. 获取文件锁。
2. reload 最新 entries。
3. 校验 + 变更 entries。
4. 原子写盘。
5. 返回 `_success_response`：含 live `entries`、`usage`（`pct% — current/limit chars`）、`entry_count`。

## 分支路径

- 扫描命中注入模式 → `success: false`，磁盘不变。
- 超 char limit → `success: false`，带 `current_entries` 供模型裁剪。
- 写盘 `RuntimeError` → 向上抛出（证据不足：主循环是否捕获并包装为 tool error 需调用方确认）。

## 失败处理

- 注入/窃密正则命中 → 明确 error 字符串，含 pattern id（如 `prompt_injection`）。
- 隐形 Unicode（U+200B 等 7 种）→ 拒绝并报告码点。
- 多 match 歧义 → 要求更具体的 `old_text`。

## 幂等性 / 一致性约束

- add 相同 content：返回 success 但不重复（近似幂等）。
- 文件锁保证同 profile 多进程 read-modify-write 安全；快照与会话内 live 故意短暂不一致。

## 代码骨架

```python
def add(store, target, content):
    content = content.strip()
    if err := scan(content):
        return fail(err)
    with file_lock(store.path(target)):
        store.reload(target)
        entries = store.entries(target)
        if content in entries:
            return ok(store, target, "Entry already exists.")
        if char_count(entries + [content]) > store.limit(target):
            return fail(f"Memory at {char_count(entries)}/{store.limit(target)}...")
        entries.append(content)
        store.set_entries(target, entries)
        store.save(target)
    return ok(store, target, "Entry added.")
```

## 最小验证清单

- add 后 `format_for_system_prompt()` 不变，响应 JSON 中 `entries` 含新条目。
- 超 limit 时 `success=false` 且磁盘无变化。
- 注入字符串 `"ignore previous instructions"` 被拒绝。
- replace 两个不同条目含同一 `old_text` 时返回 `matches` previews。
