# 阶段6：压缩后失效快照并重新加载

## 触发条件

`_compress_context()` 在阶段 5 flush 与 `on_pre_compress` 之后，完成 `context_compressor.compress()` 得到新 messages。

## 输入字段

| 字段 | 说明 |
|------|------|
| `_cached_system_prompt` | 将被置 None |
| `_memory_store` | 若存在则 `load_from_disk()` |
| `system_message` | 传入 `_build_system_prompt(system_message)` |
| session DB | 结束旧 session、创建 `parent_session_id` 链、写入新 system prompt |

## 判定规则

1. `_invalidate_system_prompt()`：
   - `_cached_system_prompt = None`
   - 若 `_memory_store`：`load_from_disk()` → 重新捕获 `_system_prompt_snapshot`（含 flush 写入）。
2. `new_system_prompt = _build_system_prompt(system_message)` → 赋给 `_cached_system_prompt`。
3. Session 拆分：`end_session(old)` → 新 `session_id` → `create_session(..., parent_session_id=old)` → `update_system_prompt(new)` → `_last_flushed_db_idx = 0`。
4. 压缩计数 `_cc >= 2` 时向用户提示考虑 `/new` 新会话。

**允许刷新快照的边界**：上下文压缩（及新会话显式重建）。普通 memory 工具写入不触发本阶段。

## 状态读写位置

- 读：磁盘 MEMORY/USER（reload）。
- 写：内存 snapshot、`_cached_system_prompt`、session DB 元数据。

## 正常路径

1. flush 写入磁盘。
2. invalidate + reload → snapshot 含新条目。
3. rebuild system prompt → 新前缀含更新记忆块。
4. 新 session 行保存新 prompt 供 gateway 续会话恢复。

## 分支路径

- 无 `_memory_store` → 只 invalidate + rebuild，无 reload。
- session DB 失败 → 压缩仍完成，warning 日志；新 session 可能不被索引。
- 无 memory 文件变化 → reload 后 snapshot 与 flush 前相同（仍合法）。

## 失败处理

- `load_from_disk` 读失败 → 按阶段 1 规则空 entries。
- DB 拆分异常 → `logger.warning`，不 rollback 压缩结果。

## 幂等性 / 一致性约束

压缩后 system prompt **必然**变化（至少摘要 user 消息 + 可能的新记忆块）。与「会话内前缀不变」策略的**唯一**例外窗口。Anthropic 缓存需接受新前缀。

## 代码骨架

```python
def after_compress(agent, messages, system_message):
    agent.flush_memories(messages, min_turns=0)
    agent.memory_manager.on_pre_compress(messages)  # 外部层，见另一方案
    compressed = agent.compressor.compress(messages)
    agent.invalidate_system_prompt()  # clears cache + memory_store.load_from_disk()
    new_prompt = agent.build_system_prompt(system_message)
    agent.cached_system_prompt = new_prompt
    agent.split_session(new_prompt)
    return compressed, new_prompt
```

## 最小验证清单

- 压缩前 add 记忆（live 有、snapshot 无）→ 压缩后新 system prompt 含该条目。
- `_cached_system_prompt is None` 仅短暂存在于 invalidate 与 rebuild 之间。
- 新 session 的 `parent_session_id` 指向旧 session。
