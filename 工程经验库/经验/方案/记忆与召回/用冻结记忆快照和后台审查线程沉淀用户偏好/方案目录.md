# 方案目录：用冻结记忆快照和后台审查线程沉淀用户偏好

> 本方案覆盖**层 1 本地文件偏好**。外部 provider 见《用提供者编排层统一外部记忆预取与同步》；情景召回见《用全文检索和摘要实现历史会话情景召回》。

## 所属方案

- [用冻结记忆快照和后台审查线程沉淀用户偏好](../用冻结记忆快照和后台审查线程沉淀用户偏好.md)

## 原子卡（按执行顺序）

1. [阶段1-记忆文件加载时冻结系统提示词快照](阶段1-记忆文件加载时冻结系统提示词快照.md) —— 加载文件、去重、捕获 `_system_prompt_snapshot`。
2. [阶段2-记忆工具写入时更新实时状态但不刷新快照](阶段2-记忆工具写入时更新实时状态但不刷新快照.md) —— add/replace/remove：锁、限长、扫描、原子写盘；**不**改快照。
3. [阶段3-系统提示词读快照而工具响应读实时条目](阶段3-系统提示词读快照而工具响应读实时条目.md) —— 双状态消费分界；模型靠 tool result 感知 live 变更。
4. [阶段4-轮次nudge计数与memory调用重置](阶段4-轮次nudge计数与memory调用重置.md) —— 每 user turn 递增；达 interval 设审查标志；memory 工具调用清零。
5. [阶段5-压缩前flush强制保存记忆](阶段5-压缩前flush强制保存记忆.md) —— **必须在阶段 6 之前**；仅 memory 工具单次 LLM；sentinel 清理。
6. [阶段6-压缩后失效快照并重新加载](阶段6-压缩后失效快照并重新加载.md) —— invalidate + `load_from_disk` + 重建 system prompt + session 父子链。
7. [阶段7-主响应交付后后台审查记忆和技能](阶段7-主响应交付后后台审查记忆和技能.md) —— 依赖阶段 4 标志；daemon 线程 max_iter=8；不阻塞 final_response。

## 阶段衔接约束

- 阶段 2/7 写入 → 阶段 3 仍读旧快照，直到阶段 6。
- 阶段 5 → 阶段 6 顺序不可颠倒。
- 阶段 4 设置的审查标志 → 仅阶段 7 消费；与外部 sync（另一方案阶段 3）并行但独立。
