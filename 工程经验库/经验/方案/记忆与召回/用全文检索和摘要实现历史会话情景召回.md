# 用全文检索和摘要实现历史会话情景召回

## 1. 问题

长期智能体积累大量历史会话。全量注入提示词成本不可控；写入本地偏好文件会污染跨会话稳定层。需要按需检索、排除当前 lineage、压缩为查询相关摘要的机制。

**与层 1 偏好存储的分界**：任务进度、会话结果、已完成工作日志 → 用本工具召回 transcript；用户偏好、环境事实 → 用 memory 工具写入偏好文件（见 schema 描述）。

## 2. 适用约束

- 消息持久化到 FTS5 可检索存储。
- 存在 compression/delegation 产生的 `parent_session_id` 链。
- 当前会话上下文已在窗口内，须排除 lineage 重复召回。
- 摘要模型可能不可用，须 raw preview 降级。

## 3. 核心思路

空查询 → recent 元数据浏览（无 LLM）。非空 → FTS 命中 → 解析根会话 → 排除 current root → 并发摘要 → 失败则 500 字符 preview。Agent 主循环拦截执行并注入 `current_session_id`。

## 4. 通用结构

- **SessionStore + FTS5**：消息全文索引。
- **LineageResolver**：沿 parent 向上，visited 防环。
- **SessionSummarizer**：辅助模型，max 3 retries。
- **RawPreviewFallback**：500 字符 + 截断标记。
- **Agent intercept**：不经 registry 分发，需 session DB。

## 5. 处理流程

| 阶段 | 内容 |
|------|------|
| 1 | 空 query → recent 列表，exclude tool source |
| 2 | FTS 50 条 → lineage 聚合 → limit≤5 |
| 3 | 加载消息 → 围绕 query 截断 100k → 并发摘要 60s |
| 4 | Agent 级拦截 + current_session_id |

## 6. 异常处理

- FTS 语法错误 → 空列表。
- DB 不可用 → tool error。
- 单会话摘要失败 → raw preview，不丢弃命中。
- 整体超时 60s → 错误 JSON 提示缩小查询。

## 7. 具体语言实现

见 `tools/session_search_tool.py` 与各阶段原子卡。

## 8. 测试点

- 空 query 零 LLM 调用。
- current lineage 命中排除。
- limit=10 入口截断为 5。
- 摘要 None 时仍有 preview 字段。
- Agent 无 session_db 时返回错误 JSON。

## 9. 适用场景 / 不适用场景

适用于跨会话复盘、个人助手情景问答。不适用于精确 SQL 审计、需完整 verbatim transcript 的合规场景。

## 10. 风险与反模式

- 不排除 lineage → 自我引用浪费窗口。
- 摘要失败静默丢弃 → 用户以为无历史。
- 把情景回忆写入 MEMORY.md → 偏好污染。
